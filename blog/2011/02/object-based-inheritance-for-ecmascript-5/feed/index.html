]]> By: Axel Rauschmayer http://www.uxebu.com/blog/2011/02/object-based-inheritance-for-ecmascript-5/#comment-936 Thu, 24 Feb 2011 00:12:41 +0000 http://uxebu.com/blog/?p=1405#comment-936 Great post! I would love for JavaScript to become purely prototypal.

Note that setting instance variables to null in the prototype leads to unnecessary properties:
js> var p = Person.create("John", "Doe");
js> Object.getOwnPropertyNames(p)
firstName,lastName
js> Object.getOwnPropertyNames(Object.getPrototypeOf(p))
firstName,lastName,_construct,toString

The prototype should only contain shared data (=methods), while _construct() adds the non-shared data (=instance variables). In that regard, how subclassing is often done in JS is also a bit of an anti-pattern:
function Sub() {}
Sub.prototype = new Super();
Now the protoype of all instances of Sub has the own properties of Super, which are never used.
