Thank you for your comment! Please allow me to answer in English. Your first question was how EmbedJS relates to dojox.mobile, and the short answer: it doesn't – but let me explain this.

Many of the existing JavaScript frameworks and toolkits that target mobile development offer ready-made UI components and widgets that mimic native UI controls both in looks and behavior. But what we experienced during our projects is that most of the applications we built (I'm almost sure even all of them) never used any UI components that looked or behaved like standard native controls. Consider a reader application for digital magazines, for example: There are quite a lot of UI components there. You have a store in which you can see magazines on a shelf with a description next to them. Inside of a magazine, you have a bottom area with some buttons that slides in when you tap on that area. You have a flipview that displays thumbnails of pages and displays story titles and descriptions and that you can slide around. You have loads of interactive elements that are animated in fancy ways. And so on…

But, none of these resemble any "standard" native UI control in any way – and that's where EmbedJS drops in. We needed a toolkit that offered us helper methods and syntactic sugar that aided us in rapid development of those custom UI components. But every existing framework out there came with the load of widgets we would never actually use – and this is why we created EmbedJS, and this is what it aims at. It takes a different approach than dojox.mobile – it's goal is to be a lightweight, optimized toolkit that provides you the basic tools you need to build an application from scratch – no matter if it's a phone, a tablet or a tv – or a fridge
