I don't think it is 100% testing framework's fault, JavaScript evaluates the arguments before calling the expect(), so the matcher has no idea what actually goes wrong. It would be nice if the testing engine at least printed the offending line using stack trace.

Programmer is expected instead to add a helpful message, duplicating the expressions as in expect(2 + 2).toEqual(4, '2 + 2 === 4'); To me this breaks do not repeat yourself principle really badly.

For now, I am using my own lazy assertions https://github.com/bahmutov/lazy-ass + helper BDD wrapper https://github.com/bahmutov/lazy-ass-helpful so that when I write

<code class="javascript">

helpDescribe('addition', function () {
it('adds numbers', function () {
lazyAss(add(2, 2) === 5); // wrong on purpose
});
});
</code>
I will get the following
module addition test adds numbers failed
condition [add(2, 2) === 5]
